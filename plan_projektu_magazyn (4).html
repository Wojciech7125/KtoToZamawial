System Zarządzania Magazynem

Plan projektu zaliczeniowego · 4 osoby · aplikacja webowa · ASP.NET Core Razor Pages + MySQL

Cel: Aplikacja webowa do zarządzania magazynem (ASP.NET Core Razor Pages, .NET 10). Zaliczenie z programowania webowego — prosty CRUD z logowaniem, nic ponad to. Każda z 4 osób (Gabryś, Adam, Krzyś, Wojtek) dostaje swoją część.
Dlaczego Razor Pages a nie MVC? Razor Pages to uproszczona wersja MVC dla prostych stron. Jedna strona = jedna para plików (.cshtml + .cshtml.cs). Ścieżka URL pokrywa się ze ścieżką w folderze — zero konfiguracji routingu. Mniej pojęć do nauczenia, mniej konfliktów w Git przy 4-osobowej grupie.

Stack technologiczny

  • .NET 10 + ASP.NET Core Razor Pages — framework webowy, jedna strona = jedna para plików
  • MySQL — baza danych (XAMPP lokalnie)
  • MySqlConnector — ręczne zapytania SQL NuGet
  • BCrypt.Net-Next — hashowanie haseł NuGet
  • QuestPDF — generowanie raportów PDF NuGet
  • Bootstrap 5 — ładny wygląd bez pisania CSS (przychodzi z szablonem)
  • Git + GitHub — wspólne repozytorium
Tylko 3 paczki NuGet. Bez Entity Framework, bez Identity, bez ViewModeli. Wszystko co się dzieje w bazie — widzicie w kodzie jako SQL. Zero magii.

Jak to zaprogramować — przewodnik krok po kroku

Konkretne wskazówki co do plików i metod. Pseudokod pokazuje strukturę — logikę piszecie sami, żeby faktycznie się czegoś nauczyć. Każdy znajdzie tu swoją część.

Jak działa Razor Pages (przeczytajcie wszyscy)

Każda strona to dwa pliki:

  • Pages/Products/Index.cshtml — HTML z kawałkami C# (to co widzi user)
  • Pages/Products/Index.cshtml.cs — klasa C# z logiką tej strony

URL /Products/Index automatycznie trafia do tej pary plików. URL /Products/Edit?id=5 → Pages/Products/Edit.cshtml.

W pliku .cs klasa dziedziczy z PageModel i ma metody OnGet() (gdy ktoś wchodzi) i OnPost() (gdy wyśle formularz). Parametry z URL są automatycznie mapowane na pola klasy z atrybutem [BindProperty].

Najprostszy możliwy przykład:

// Pages/Test.cshtml.cs public class TestModel : PageModel { public string Wiadomosc { get; set; } public void OnGet() { Wiadomosc = "Cześć!"; } }
<!-- Pages/Test.cshtml --> @page @model TestModel <h1>@Model.Wiadomosc</h1>

Wejście na /Test pokaże "Cześć!". To cała filozofia.

Db.cs — helper SQL (Gabryś)

using System.Data; using MySqlConnector; public class Db { private readonly string _cs; public Db(IConfiguration cfg) { _cs = cfg.GetConnectionString("MySQL"); } // SELECT - dla list i wyników public DataTable Query(string sql, params MySqlParameter[] p) { using var conn = new MySqlConnection(_cs); conn.Open(); using var cmd = new MySqlCommand(sql, conn); cmd.Parameters.AddRange(p); var dt = new DataTable(); dt.Load(cmd.ExecuteReader()); return dt; } // INSERT / UPDATE / DELETE - zwraca liczbę zmienionych wierszy public int Exec(string sql, params MySqlParameter[] p) { using var conn = new MySqlConnection(_cs); conn.Open(); using var cmd = new MySqlCommand(sql, conn); cmd.Parameters.AddRange(p); return cmd.ExecuteNonQuery(); } // SELECT zwracający jedną wartość (COUNT, pojedynczy ID, itd.) public object Scalar(string sql, params MySqlParameter[] p) { using var conn = new MySqlConnection(_cs); conn.Open(); using var cmd = new MySqlCommand(sql, conn); cmd.Parameters.AddRange(p); return cmd.ExecuteScalar(); } }

Używanie w stronie: w konstruktorze PageModel przyjmujemy Db db — ASP.NET wstrzyknie sam.

ZAWSZE parametryzowane zapytania! Nigdy nie sklejaj SQL-a ze stringów ("WHERE id = " + userInput) — to luka SQL Injection. Używaj @id i new MySqlParameter("@id", id).

RequireLoginAttribute.cs — ochrona stron (Gabryś)

using Microsoft.AspNetCore.Mvc; using Microsoft.AspNetCore.Mvc.Filters; public class RequireLoginAttribute : ActionFilterAttribute { public override void OnPageHandlerExecuting(PageHandlerExecutingContext ctx) { if (ctx.HttpContext.Session.GetInt32("UserId") == null) { ctx.Result = new RedirectToPageResult("/Account/Login"); } } }

Używanie: dopisujesz [RequireLogin] nad klasą PageModel. Jeśli user nie zalogowany — przekierowanie na login.

Gabryś — Fundamenty, Logowanie, Panel admina

1 Schemat bazy — database.sql

CREATE DATABASE magazyn CHARACTER SET utf8mb4; USE magazyn; CREATE TABLE users ( id INT AUTO_INCREMENT PRIMARY KEY, login VARCHAR(50) UNIQUE NOT NULL, password_hash VARCHAR(255) NOT NULL, role VARCHAR(20) DEFAULT 'pracownik', created_at DATETIME DEFAULT CURRENT_TIMESTAMP ); CREATE TABLE categories ( id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(100) UNIQUE NOT NULL ); CREATE TABLE products ( id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(200) NOT NULL, sku VARCHAR(50) UNIQUE NOT NULL, category_id INT, quantity INT DEFAULT 0, min_quantity INT DEFAULT 0, unit VARCHAR(20) DEFAULT 'szt.', price DECIMAL(10,2) DEFAULT 0, FOREIGN KEY (category_id) REFERENCES categories(id) ); CREATE TABLE operations ( id INT AUTO_INCREMENT PRIMARY KEY, product_id INT NOT NULL, user_id INT NOT NULL, type ENUM('IN', 'OUT') NOT NULL, quantity INT NOT NULL, note VARCHAR(255), created_at DATETIME DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (product_id) REFERENCES products(id), FOREIGN KEY (user_id) REFERENCES users(id) ); -- Domyślny admin (login: admin, hasło: admin123) -- Hash wygenerujcie w C#: BCrypt.Net.BCrypt.HashPassword("admin123") -- i wklejcie tutaj: INSERT INTO users (login, password_hash, role) VALUES ('admin', 'TUTAJ_HASH', 'admin');
Wygenerowanie hasha: zrób mały programik konsolowy z linijką Console.WriteLine(BCrypt.Net.BCrypt.HashPassword("admin123")) albo odpal raz w aplikacji i podejrzyj w debuggerze. Wstaw wynik do INSERTa.

// Operation.cshtml.cs [RequireLogin] public class OperationModel : PageModel { private readonly Db _db; public OperationModel(Db db) { _db = db; } [BindProperty] public int ProductId { get; set; } [BindProperty] public int Quantity { get; set; } [BindProperty] public string Note { get; set; } [BindProperty(SupportsGet = true)] public string Type { get; set; } = "IN"; public DataTable Produkty { get; set; } public string Blad { get; set; } public void OnGet() { Produkty = _db.Query("SELECT id, name, quantity FROM products ORDER BY name"); } public IActionResult OnPost() { // Dla wydania - sprawdź czy wystarczy if (Type == "OUT") { var obecna = Convert.ToInt32(_db.Scalar( "SELECT quantity FROM products WHERE id = @id", new MySqlParameter("@id", ProductId))); if (obecna < Quantity) { Blad = "Za mało w magazynie!"; OnGet(); return Page(); } } // 1. Zmień stan produktu string znak = Type == "IN" ? "+" : "-"; _db.Exec( $"UPDATE products SET quantity = quantity {znak} @q WHERE id = @id", new MySqlParameter("@q", Quantity), new MySqlParameter("@id", ProductId)); // 2. Zapisz do historii _db.Exec(@" INSERT INTO operations (product_id, user_id, type, quantity, note) VALUES (@p, @u, @t, @q, @n)", new MySqlParameter("@p", ProductId), new MySqlParameter("@u", HttpContext.Session.GetInt32("UserId")), new MySqlParameter("@t", Type), new MySqlParameter("@q", Quantity), new MySqlParameter("@n", Note ?? DateTime.Today.AddDays(1); Operacje = _db.Query(@" SELECT o.created_at, o.type, o.quantity, o.note, p.name AS product_name, u.login AS user_login FROM operations o JOIN products p ON p.id = o.product_id JOIN users u ON u.id = o.user_id WHERE o.created_at BETWEEN @od AND @do AND (@typ = '' OR o.type = @typ) ORDER BY o.created_at DESC", new MySqlParameter("@od", od), new MySqlParameter("@do", do_), new MySqlParameter("@typ", Typ ?? "")); } }

W widoku: formularz GET z <input type="date" name="Od">, <input type="date" name="Do">, <select name="Typ">. Po kliknięciu "Filtruj" → nowe parametry w URL → OnGet wywołane z nowymi wartościami.

Typowe błędy których unikać

  • Konkatenacja stringów w SQL — "WHERE id = " + id → SQL Injection. Zawsze parametry @id i new MySqlParameter(...).
  • Zapomnienie [RequireLogin] na stronie — każdy bez logowania może wejść.
  • Zapisywanie haseł plaintextem — zawsze BCrypt.Net.BCrypt.HashPassword().
  • Brak app.UseSession() w Program.cs — logowanie nie będzie działać, sesja nie zadziała.
  • Commit appsettings.Development.json z hasłem — do .gitignore.
  • Brak OnGet() w modelu strony — strona wyrzuci błąd przy wejściu.
  • Edycja _Layout.cshtml przez 2 osoby naraz — merge conflict pewny. Uzgadniajcie.
  • Nazwa strony nie pasuje do klasy — plik Index.cshtml.cs musi mieć klasę IndexModel. Inaczej błąd.

Pytania do ustalenia przed kodowaniem

  • Gdzie stawiamy MySQL? Każdy lokalnie (XAMPP) czy jeden wspólny serwer?
